# System Instructions: AML Fraud Investigation Platform

## Project Overview

Build an **Anti-Money Laundering (AML) Investigation Platform** designed to detect, analyze, and visualize suspicious financial activity.
The system enables investigators and analysts to monitor account behavior, identify money laundering patterns, track fund movement across accounts, calculate risk scores, and generate investigation reports.

The platform integrates graph analysis, rule-based fraud detection, anomaly scoring, and interactive visual investigation tools to simulate a real banking fraud detection system.

Primary capabilities include:

* Detection of suspicious transaction patterns
* Risk scoring for accounts and networks
* Graph-based fund flow investigation
* Timeline analysis of transaction activity
* Fraud alerts and heat map visualization
* AI-assisted explanation of suspicious behavior
* Automated evidence report generation

Four primary cards display the overall system risk indicators.

### Card 1: Total Monitored Accounts

Displays the total number of accounts currently tracked by the system.

Includes:

* Number of active accounts
* Number of flagged accounts
* Percentage change in flagged accounts compared to the previous period

### Card 2: Suspicious Transactions

### Card 3: High Risk Accounts

Alerts may include:

* Rapid layering

# Investigation Graph Section

This is the **core feature of the AML platform**.

The system visualizes financial relationships between accounts using a graph structure powered by
Cytoscape.js and data stored in
Neo4j.

### Graph Capabilities

The graph displays:

* Accounts as nodes
* Transactions as edges
* Transaction amounts as labels
* Suspicious nodes highlighted with color indicators

Node color meaning:

Blue → Normal account
Yellow → Medium risk account
Red → High risk account

Graph interactions allow investigators to:

* Expand transaction paths
* Click nodes to reveal connected accounts
* Highlight suspicious transaction chains
* Investigate laundering paths through intermediary accounts

Example laundering path:

Account A1 → Account B1 → Account C1 → Account D1

# Risk Score Panel

Displays the **calculated risk score for the selected account**.

The risk score ranges from 0 to 100 and is determined using fraud pattern severity and anomaly indicators.

Example:

Account Risk Score: 82 / 100

Risk classification levels:

Low Risk: 0–30
Medium Risk: 31–60
High Risk: 61–100

The panel also shows the specific fraud patterns contributing to the score.

---

# Transaction Timeline Panel

The timeline provides a chronological view of transactions involving the investigated account.

Displayed information includes:

Transaction time
Source account
Destination account
Transaction amount

Example timeline:

10:01 — A1 → B1 — ₹90,000
10:03 — B1 → C1 — ₹89,000
10:05 — C1 → D1 — ₹88,000

This visualization helps investigators identify **rapid layering behavior**.
